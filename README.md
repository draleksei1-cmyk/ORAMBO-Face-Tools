# ORAMBO Face Tools 0.1.2

Набор безопасных поэтапных инструментов SketchUp для подготовки импортированных DWG-контуров к созданию граней.

## Установка

1. Откройте SketchUp 2021–2026.
2. Откройте **Extensions → Extension Manager**.
3. Нажмите **Install Extension** и выберите `ORAMBO_Face_Tools_0.1.2.rbz`.
4. Если панель не появилась, включите её через **View → Toolbars → ORAMBO Face Tools**.

## Рекомендуемый порядок

Для нормального DWG сначала используйте мягкий сценарий:

1. `Flatten Edges To Z`
2. `Make Faces`

Если контуры не собираются, используйте ремонтный сценарий:

1. `Break To Segments`
2. `Flatten Edges To Z`
3. `Make Faces`

`Break To Segments` разрушает структуру вложенных групп и компонентов. Плагин всегда запрашивает подтверждение. Каждый инструмент отменяется одним нажатием Undo.

## Инструменты

### Break To Segments

Собирает выбранные объекты в рабочую группу, ограниченно раскрывает вложения и может преобразовать Curve/Arc в обычные сегменты или удалить совпадающие рёбра. Переключатель преобразования кривых включён по умолчанию, но его можно отключить. Компоненты перед изменением становятся уникальными. Отражённые компоненты требуют отдельного подтверждения.

### Flatten Edges To Z

Меняет мировую Z-координату вершин, а не перемещает группу целиком. Корректно учитывает смещение, поворот и вложенность контейнеров. После обработки сообщает остаточный разброс Z и предлагает удалить найденные микрорёбра.

### Make Faces

Проверяет плоскостность, закрывает небольшие зазоры с жёстким лимитом и создаёт грани через SketchUp `find_faces`. Замыкающие рёбра получают тег `ORAMBO_Gap_Closers`. Новые грани можно ориентировать к камере или вверх по мировой Z.

### Select Open Ends

Находит вершины степени 1, выделяет проблемные рёбра и показывает свободные концы красными крестиками в отдельной группе `ORAMBO_Open_Ends`.

### Highlight Gaps

Находит ближайшие пары свободных концов в заданной дистанции и показывает разрывы красными линиями в отдельной группе `ORAMBO_Gaps`, не изменяя исходные контуры.

## Безопасность

- Обрабатывается только выделение.
- Скрытые и заблокированные объекты по умолчанию пропускаются.
- При 50 000 и более рёбер появляется предупреждение.
- Закрывается не более 5 000 зазоров за один запуск.
- Выполняется не более 50 проходов раскрытия вложений.
- Подробные ошибки печатаются в **Window → Ruby Console**.

Перед обработкой важного DWG сохраните модель. Undo рассчитан на один шаг, но резервная копия всё равно дешевле архитектурной археологии.

## Автоматические обновления

Плагин один раз проверяет публичные [GitHub Releases](https://github.com/draleksei1-cmyk/ORAMBO-Face-Tools/releases) через несколько секунд после запуска. Если доступна новая стабильная версия, SketchUp показывает нативную плашку с кнопками **Обновить** и **Позже**.

Обновление сначала целиком скачивается во временную папку. Пути и SHA-256 каждого файла проверяются до изменения установленного плагина. При ошибке замены восстанавливается предыдущая версия.

Обычные обновления Ruby-логики применяются сразу без переустановки RBZ и перезапуска SketchUp. Если релиз меняет загрузчик, меню или структуру панели, плагин установит файлы и попросит перезапустить SketchUp: уже созданные объекты интерфейса Ruby API безопасно выгрузить не позволяет.

Ручная проверка доступна через **Extensions → ORAMBO Face Tools → Check for Updates**. Ошибки автоматической проверки сети не мешают работе плагина и выводятся только в Ruby Console.

## Ручная проверка

Откройте **Window → Ruby Console** и загрузите файл `sketchup_test/manual_acceptance.rb` из репозитория:

```ruby
load 'C:/полный/путь/sketchup_test/manual_acceptance.rb'
ORAMBO::FaceTools::ManualAcceptance.square
```

Доступные сценарии: `square`, `small_gap`, `large_gap`, `noncoplanar`, `nested_rotated`, `component_instances`, `curve`, `locked_and_hidden`, `mirrored_component`.

Ожидания:

- `square` + Make Faces → одна новая грань;
- `small_gap` при допуске 1 мм → один замыкатель и одна грань;
- `large_gap` при допуске 1 мм → грань не создаётся;
- `noncoplanar` + Make Faces → предупреждение о Z, без грани;
- `nested_rotated` + Flatten на Z=0 → мировая Z всех рёбер равна нулю;
- `component_instances` → меняется только выбранный экземпляр;
- `curve` + Break с отключённым преобразованием → дуга остаётся кривой;
- `locked_and_hidden` → объекты пропускаются и попадают в отчёт;
- `mirrored_component` + Break → появляется отдельное подтверждение.

## Разработка

Автотесты рассчитаны на Ruby 2.7. Сборка RBZ:

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build_rbz.ps1
```

Автотесты проверяют чистые алгоритмы и регистрацию UI. Геометрические интеграционные сценарии выполняются внутри SketchUp, поскольку SketchUp Ruby API не доступен отдельному системному Ruby.

### Публикация релиза

1. Обновите `EXTENSION_VERSION` в `src/orambo_face_tools.rb` и `src/orambo_face_tools/main.rb`.
2. Для изменений загрузчика или UI установите `restart_required` в `true` внутри `.github/release-config.json`.
3. Создайте и отправьте тег `vX.Y.Z`.

GitHub Actions запустит тесты, соберёт RBZ, сформирует `update_manifest.json` и опубликует оба файла в GitHub Release.
