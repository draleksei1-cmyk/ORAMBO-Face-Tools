# ORAMBO Face Tools 0.1.2 — диагностика открытых концов и зазоров

## Цель

Версия 0.1.2 должна исправить реальное разбиение SketchUp Curve/Arc, добавить две отдельные диагностические команды `Select Open Ends` и `Highlight Gaps`, а также явно показывать успешное завершение автообновления в строке состояния SketchUp.

## Разбиение кривых

`Break To Segments` сохраняет включённый по умолчанию переключатель преобразования кривых. Вместо удаления и повторного создания сегментов команда вызывает официальный метод `Sketchup::Edge#explode_curve` один раз для каждой уникальной Curve/Arc. Нативный метод снимает связь кривой, не изменяя положение её рёбер. Скрытые кривые пропускаются, если пользователь не включил обработку скрытой геометрии. Ошибка отдельной кривой записывается в отчёт и не останавливает остальные.

## Select Open Ends

Команда работает с выделенными рёбрами, группами и компонентами через существующий глубокий сборщик контекстов. Для каждого контекста она считает степень каждой вершины. Вершина со степенью 1 считается свободным концом.

Результат:

- смежные со свободными концами исходные рёбра добавляются в текущее выделение;
- в отдельной служебной группе создаются красные крестики, центрированные в мировых координатах свободных концов;
- итоговый отчёт показывает количество найденных свободных концов.

## Highlight Gaps

Команда использует те же свободные концы и спрашивает максимальную дистанцию поиска, по умолчанию 10 мм. Пространственная сетка ищет ближайшие взаимоисключающие пары в пределах дистанции, не соединяет уже связанные вершины и не использует один конец дважды.

Для каждой найденной пары в служебной группе создаётся красный отрезок между мировыми координатами концов. Это диагностическая линия, а не замыкание исходного контура: она не участвует в `find_faces` и не меняет исходную DWG-геометрию. Отчёт показывает количество пар и непарных открытых концов.

## Служебная группа

Обе команды создают геометрию только внутри отдельных групп `ORAMBO_Open_Ends` и `ORAMBO_Gaps` в текущем активном контексте и назначают им общий тег `ORAMBO_Diagnostics`. Группы получают красный материал `ORAMBO_Diagnostics_Red`.

Перед запуском команда удаляет и создаёт заново только свою группу. Поэтому крестики открытых концов и линии зазоров можно показывать одновременно, а старые результаты одного типа не смешиваются с новыми. Вся операция обёрнута одним SketchUp Undo. Исходные рёбра, группы и компоненты не изменяются.

Полуразмер красного крестика равен 0,2% диагонали bounds выбранной геометрии и ограничен диапазоном 10–500 мм. Линии крестиков и зазоров находятся внутри отдельных групп, поэтому не пересекаются топологически с исходными контурами.

## Панель и меню

После существующих трёх команд добавляются две отдельные команды:

1. `Select Open Ends` — иконка открытого узла;
2. `Highlight Gaps` — иконка двух концов с разрывом.

Обе команды доступны и на панели, и в `Extensions → ORAMBO Face Tools`. Новые минималистичные иконки соответствуют существующей синей графике панели. При горячем обновлении diagnostics добавляет две кнопки в уже существующую панель без её повторного создания; полное меню с новыми пунктами гарантированно формируется при следующем обычном запуске SketchUp.

## Сообщение автообновления

После успешной установки и горячей перезагрузки updater сохраняет существующую нативную плашку и дополнительно устанавливает:

```ruby
Sketchup.status_text = "ORAMBO Face Tools: обновление 0.1.2 успешно"
```

Для будущих версий номер подставляется из manifest. При необходимости перезапуска строка состояния сообщает, что файлы установлены и требуется перезапустить SketchUp. Ошибки обновления также получают короткий текст в строке состояния.

## Структура

- новый `diagnostics.rb` содержит чистый поиск открытых концов/пар и SketchUp-адаптер создания маркеров;
- `toolbar.rb` регистрирует две новые команды;
- `main.rb` загружает diagnostics до toolbar;
- `updater.rb` включает diagnostics в порядок hot-reload и обновляет строку состояния;
- `break_to_segments.rb` использует `Edge#explode_curve`;
- четыре новых PNG-файла содержат иконки 16/24 px.

## Тестирование

Автоматические тесты Ruby 2.7 должны подтвердить:

- каждая уникальная кривая получает ровно один вызов `explode_curve`;
- скрытая кривая пропускается при выключенной обработке hidden;
- степень вершины 1 определяется как открытый конец;
- замкнутый контур не имеет открытых концов;
- поиск gap-пар выбирает ближайшие взаимоисключающие пары и соблюдает дистанцию;
- повторный запуск заменяет старую диагностическую группу;
- toolbar и меню содержат пять рабочих команд;
- updater пишет успешный текст в `Sketchup.status_text`.

Ручная проверка выполняется на проблемном DWG пользователя: Curve должна стать набором обычных рёбер; `Select Open Ends` должен показать точные красные крестики; `Highlight Gaps` — красные линии только между ближайшими концами в пределах заданной дистанции; обновление 0.1.1 → 0.1.2 должно завершиться видимой строкой состояния.

## Релиз

Версия повышается до 0.1.2. Изменения публикуются тегом `v0.1.2` с `restart_required: false`. Новый `diagnostics.rb` входит в hot-reload и после загрузки идемпотентно добавляет недостающие кнопки в существующую панель. Поэтому исправление кривых, строка успешного обновления и обе диагностические кнопки применяются без переустановки RBZ и без обязательного перезапуска SketchUp.
