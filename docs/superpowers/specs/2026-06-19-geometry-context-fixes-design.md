# ORAMBO Face Tools 0.1.1 — исправление геометрических контекстов

## Цель

Исправить три подтверждённые пользовательским тестированием проблемы:

1. Break To Segments оставляет SketchUp-кривые связанными;
2. Flatten Edges To Z не перемещает вершины и завершается ошибкой о `transform_by_vectors`;
3. Make Faces не создаёт грани и завершается `NoMethodError` для `intersect_with`.

Версия 0.1.1 должна устанавливаться существующим механизмом автообновления GitHub без повторной установки RBZ.

## Корневая причина

`Utils.collect_edge_contexts` записывает `entity.parent` в поле `context[:entities]`. Для ребра верхнего уровня `entity.parent` является `Sketchup::Model`, а для ребра внутри группы или компонента — `Sketchup::ComponentDefinition`. Методы `intersect_with`, `grep`, `add_line` и `transform_by_vectors` принадлежат `Sketchup::Entities`, поэтому Make Faces и Flatten получают объект неверного типа и останавливаются.

Break To Segments имеет отдельную проблему поведения: переключатель преобразования кривых существует, но выключен по умолчанию. Это не соответствует ожидаемому назначению первого инструмента.

## Выбранная архитектура

Исправление выполняется централизованно в `Utils.collect_edge_contexts`. Обход должен передавать фактическую коллекцию `Sketchup::Entities` явно:

- для непосредственно выделенных рёбер — `model.active_entities`;
- для содержимого группы или компонента — `entity.definition.entities`;
- ключом объединения контекста становится `entities.object_id`, а не `entity.parent.object_id`.

Так один исправленный контракт обслуживает Make Faces и Flatten без отдельных обходных путей.

## Поведение инструментов

### Break To Segments

Переключатель «Кривые и дуги перевести в отдельные отрезки» сохраняется и включается по умолчанию. При отключении кривые остаются связанными. При включении каждая выбранная кривая или дуга заменяется обычными рёбрами с сохранением слоя, материала, видимости, сглаживания и геометрии сегментов.

### Flatten Edges To Z

Инструмент получает настоящий `Sketchup::Entities` и перемещает все уникальные вершины через один вызов `transform_by_vectors`. Целевая высота остаётся мировой Z, включая геометрию во вложенных трансформированных группах и уникализированных компонентах. После операции выполняется существующая проверка остаточного разброса Z и микрорёбер.

### Make Faces

Для каждого контекста используется последовательность:

1. получить актуальные допустимые рёбра;
2. при включённой опции разбить пересечения через `entities.intersect_with`;
3. заново получить актуальные рёбра;
4. добавить ограниченное число замыкающих линий для малых щелей;
5. снова получить актуальные рёбра;
6. вызвать `find_faces` для каждого валидного ребра;
7. определить и ориентировать только вновь созданные грани, если пользователь не запросил обработку существующих.

Ошибка отдельного ребра или необязательного разбиения пересечений записывается в предупреждения и не останавливает остальные контексты.

## Тестирование

Регрессионные тесты сначала должны воспроизвести ошибки старой версии:

- контекст рёбер верхнего уровня содержит `model.active_entities`, а не модель;
- контекст вложенной группы содержит `definition.entities`, а не определение;
- Flatten вызывает `transform_by_vectors` у коллекции Entities и передаёт одинаковое число вершин и векторов;
- Make Faces вызывает `intersect_with`, `grep`, `add_line` и `find_faces` через коллекцию Entities;
- переключатель преобразования кривых включён по умолчанию;
- отключённый переключатель не преобразует кривые.

После модульных тестов выполняются проверка Ruby-синтаксиса, сборка RBZ, проверка содержимого RBZ и манифеста обновления. Ручная проверка в SketchUp использует тот же DWG-фрагмент пользователя: кривые должны стать отрезками, все вершины — лечь на выбранную мировую Z, а замкнутые контуры — получить грани без критической ошибки.

## Релиз

Версия расширения меняется с `0.1.0` на `0.1.1`. После зелёных проверок создаётся тег `v0.1.1`. GitHub Actions публикует новый RBZ и `update_manifest.json`; установленная версия 0.1.0 обнаруживает релиз и предлагает обновление. Изменяемые файлы входят в список горячей перезагрузки, поэтому штатное обновление применяется без переустановки RBZ и перезапуска SketchUp.

## Вне области исправления

- изменение внешнего вида панели инструментов;
- автоматическое удаление геометрии, не относящейся к кривым;
- изменение пользовательских лимитов замыкания щелей;
- создание собственного геометрического движка вместо API SketchUp.
