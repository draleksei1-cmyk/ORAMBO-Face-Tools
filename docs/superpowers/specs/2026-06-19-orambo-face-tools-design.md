# ORAMBO Face Tools 0.1.0 — проектное решение

## Цель

Создать устанавливаемое расширение SketchUp 2021–2026 для поэтапной подготовки импортированной DWG-геометрии к созданию граней. Расширение предоставляет три независимые команды: `Break To Segments`, `Flatten Edges To Z` и `Make Faces`.

Исходное техническое задание: `C:/Users/drale/Downloads/orambo_face_tools_tz.md`.

## Принятый подход

Используется модульная архитектура на нативном SketchUp Ruby API без сторонних gem и зависимостей от других расширений. Названия команд остаются английскими, а подтверждения, ошибки и итоговые отчёты выводятся на русском языке.

Версия 0.1.0 не получает HtmlDialog, объединённый конвейер из трёх команд или автоматическое исправление всей модели. Пользователь запускает и контролирует каждый этап отдельно.

## Поставка

Проект содержит:

- стандартный загрузчик расширения;
- каталог модулей `orambo_face_tools`;
- нативное меню `Extensions → ORAMBO Face Tools`;
- панель с тремя минималистичными кнопками;
- PNG-иконки 16×16 и 24×24;
- автоматические тесты независимой вычислительной логики;
- Ruby-сценарий ручной проверки внутри SketchUp;
- инструкцию по установке и проверке;
- готовый архив `ORAMBO_Face_Tools_0.1.0.rbz`.

## Архитектура

### Загрузка и интерфейс

`orambo_face_tools.rb` регистрирует `SketchupExtension`. `main.rb` загружает внутренние модули один раз. `toolbar.rb` создаёт меню и панель. Отсутствующая иконка не должна мешать регистрации команды.

### Общие службы

- `utils.rb`: обход выделения и вложенных контейнеров, преобразования координат, подсчёты, сеточные ключи, определение отражённых трансформаций и чистые алгоритмы.
- `safety.rb`: предварительные проверки, подтверждения опасных операций, лимиты, работа с locked/hidden-объектами и единая оболочка Undo.
- `progress.rb`: ограниченное по частоте обновление строки состояния и восстановление исходного текста.
- `report.rb`: накопление счётчиков и предупреждений, подробности в Ruby Console и краткий русский итоговый диалог.

Каждый модуль имеет одну ответственность и общается с остальными через небольшие публичные методы.

## Поток обработки

### Break To Segments

1. Проверить модель, выделение, блокировки, скрытые объекты и объём геометрии.
2. Показать явное предупреждение о разрушительной операции и параметры обработки.
3. Собрать несколько выбранных объектов в одну рабочую группу.
4. Ограниченно раскрывать вложенные группы и уникализированные компоненты.
5. По выбору преобразовать Curve/Arc в отдельные рёбра без изменения формы сегментов.
6. По выбору удалить совпадающие рёбра, считая A–B и B–A одинаковыми.
7. Оставить рабочую группу выделенной и показать отчёт.

### Flatten Edges To Z

1. Рекурсивно собрать контексты рёбер выбранной геометрии.
2. Сделать изменяемые экземпляры компонентов уникальными.
3. Для каждой вершины вычислить мировую точку через полный transform.
4. Заменить мировую Z на целевой уровень и преобразовать точку обратно в локальные координаты контейнера.
5. Проверить остаточный разброс Z и появившиеся микрорёбра.
6. Не удалять микрорёбра без отдельного подтверждения.
7. Сохранить исходный контейнер выделенным и показать отчёт.

### Make Faces

1. Собрать допустимые рёбра по отдельным `Entities`-контекстам.
2. Проверить Z-разброс до изменения геометрии; неплоские контексты пропустить с предупреждением.
3. Найти свободные концы пространственной сеткой и добавить не более `MAX_GAP_CLOSERS` коротких замыкающих рёбер.
4. Назначить замыкающим рёбрам тег `ORAMBO_Gap_Closers`.
5. Вызвать `edge.find_faces` без разрезания пересечений по умолчанию.
6. Определить только новые грани и развернуть их к камере либо вверх по мировой Z согласно настройке команды.
7. Оставить рабочий контейнер выделенным и показать отчёт.

## Трансформации и компоненты

Обход хранит `Entities`, рёбра, накопленный transform и контейнер. Все проверки высоты выполняются в мировых координатах. Изменение вершины внутри вложения выполняется через обратное преобразование в локальную систему.

Компонент делается уникальным только тогда, когда его геометрия действительно будет изменяться. Отражённый или вырожденный transform определяется заранее; опасная операция требует подтверждения либо контекст пропускается с понятным предупреждением.

## Безопасность и производительность

- Каждая команда выполняется одной операцией SketchUp и откатывается одним Undo.
- Критическое исключение вызывает `abort_operation`; ошибки отдельных элементов накапливаются без молчаливого падения команды.
- Locked, hidden и hidden edges по умолчанию пропускаются.
- Тяжёлые операции предваряются подсчётом и подтверждением.
- Взрыв вложений, замыкание разрывов, предупреждения и другие потенциально неограниченные проходы имеют жёсткие лимиты.
- Строка состояния обновляется порциями, чтобы не замедлять каждый обрабатываемый элемент.
- Подробные предупреждения печатаются в Ruby Console; диалог показывает ограниченное число строк и общие счётчики.

## Тестирование

Автоматические тесты запускаются обычным Ruby без SketchUp и проверяют чистую логику: нормализацию рёбер, пространственные ключи, поиск пар свободных концов, лимиты, математику трансформаций через тестовые адаптеры и форматирование отчёта.

Интеграционный Ruby-сценарий запускается внутри SketchUp 2026 и создаёт геометрию для сценариев из ТЗ: квадрат, разрывы 0,5 и 5 мм, вложенные и повёрнутые контейнеры, несколько экземпляров компонента, кривые, дубли, hidden/locked-объекты и mirrored transform.

Финальная приёмка включает установку RBZ, наличие меню и панели, один Undo на команду и визуальную проверку на реальном импортированном DWG.

## Границы версии 0.1.0

Не входят: HtmlDialog, Cancel во время обработки, пакетный запуск трёх этапов, сложная реконструкция полигонов, автоматическое исправление всей модели, агрессивное удаление микрорёбер и внешние зависимости.

## Критерий готовности

Версия готова, когда собирается устанавливаемый RBZ, проходит автоматические тесты, загружается в SketchUp 2026 без ошибок, выполняет сценарии ручного теста и соответствует 25 критериям приёмки исходного ТЗ. Ограничения фактической проверки на SketchUp 2021–2025 документируются; код использует только API, доступный начиная с SketchUp 2021.
